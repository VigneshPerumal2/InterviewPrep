# Research Sources

## What This Means

This page lists the public sources used to shape the prep material. The content in this workspace is summarized and adapted for interview preparation. It is not copied from those sources.

## Official Visa Context

- [Visa Payments Processing](https://developer.visa.com/capabilities/vpp): card-not-present payment processing concepts and API categories such as authorization, capture, sale, refund, void, and verification.
- [Visa Direct Guides](https://developer.visa.com/products/visa_direct/guides): push payment and pull-funding concepts such as OCT and AFT.
- [Visa Token Service Provisioning](https://developer.visa.com/capabilities/token-service-provisioning): tokenization concepts for replacing sensitive card data with payment tokens.
- [Visa Developer Center](https://developer.visa.com/): public developer documentation for Visa APIs and capabilities.

## How To Use Sources

Use official Visa docs to understand payment terminology. Use interview-report sources only as directional signals, because interview loops can vary by team, location, seniority, and hiring manager.

## Interview Answer

> I used official Visa developer documentation to understand payment capabilities and public interview reports to prepare for likely coding, design, and behavioral rounds. I understand the exact interview format can vary, so I am preparing fundamentals and payment-relevant system design rather than memorizing one fixed process.

## Common Mistakes

- Treating public interview reports as guaranteed.
- Copying answers instead of understanding concepts.
- Ignoring official payment terminology.
- Over-preparing rare questions before core fundamentals.
